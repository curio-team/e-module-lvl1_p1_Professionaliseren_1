In deze praktische afronding laat je zien dat je de volledige Git-workflow zelfstandig beheerst — van een nette mappenstructuur tot code die online op GitHub staat.

## Opdracht

Maak een nieuwe **private** repository op GitHub met een naam naar keuze. Clone deze naar een logische plek in je mappenstructuur, voeg een klein HTML-bestandje toe, en zet je code online via een commit en een push. Werk daarna nog één wijziging in, met een **tweede** commit en push.

## Stappenplan

1. Maak een private repository op GitHub.
2. Clone de repository naar je laptop (op de juiste plek in je `school`-mappenstructuur).
3. Controleer dat de verborgen `.git`-folder aanwezig is.
4. Voeg een `index.html` toe met een korte tekst over jezelf.
5. Maak een commit met een logische titel en push deze.
6. Pas de pagina aan (voeg bijvoorbeeld een lijstje toe), maak een **tweede** commit met een logische titel en push opnieuw.
7. Controleer je commitgeschiedenis op GitHub.

## Beoordeling

Je beheerst de praktijk als je kunt aantonen dat:

- de repository private is en correct gecloond is;
- er minstens **twee commits** met logische titels zijn;
- de code zichtbaar online op GitHub staat;
- je je commitgeschiedenis kunt terugvinden en uitleggen.
